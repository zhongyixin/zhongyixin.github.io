PATA1076 Forwards on Weibo

发表于 2018-09-04 | 分类于 PAT

Weibo is known as the Chinese version of Twitter. One user on Weibo may have many followers, and may follow many other users as well. Hence a social network is formed with followers relations. When a user makes a post on Weibo, all his/her followers can view and forward his/her post, which can then be forwarded again by their followers. Now given a social network, you are supposed to calculate the maximum potential amount of forwards for any specific user, assuming that only L levels of indirect followers are counted.

Input Specification:

Each input file contains one test case. For each case, the first line contains 2 positive integers: N (≤1000), the number of users; and L (≤6), the number of levels of indirect followers that are counted. Hence it is assumed that all the users are numbered from 1 to N. Then N lines follow, each in the format:

where Mi is the total number of people that user[i] follows; and user_list[i] is a list of the M[i] users that followed by user[i]. It is guaranteed that no one can follow oneself. All the numbers are separated by a space.
Then finally a positive K is given, followed by K UserID’s for query.

Output Specification:

For each UserID, you are supposed to print in one line the maximum potential amount of forwards this user can triger, assuming that everyone who can view the initial post will forward it once, and that only L levels of indirect followers are counted.

题目大意：
给出1-N号微博用户的关注名单，给出K个查询，问当前查询编号的用户L层转发下的最大转发次数
思路：
可以用邻接表记录关注每个用户的用户编号，此处需注意转发的方向为被关注用户->关注该用户的用户，输入时邻接表push的时候要注意方向~而后层序遍历并记录层数，当层数超过限定值则不再入队，最后返回总人数即可

其他注意事项：
此题中我采用了一维数组记录当前编号的层数，由于每次查询会进行覆盖所以不用预先置零。很多其他题解中给出的方法是建立结构体变量node记录当前用户的id及层数layer，其遍历过程核心代码如下,其中的next.layer在判断是否已访问前已经更新，而结果并不受影响，是因为每一个结点单独开辟空间存放层号，前后互不干扰。

但是如果采用一维数组存放layer必须先判断结点是否满足条件再进行更新，因为由于双向边的存在，如果先更新层号会导致已被访问的结点层号被抬高，后续满足条件的结点会因为层号重复计算超出L而无法计入结果。
以样例中的查询编号6为例：
第一次while()：3，同时layer[3]=1;
第二次while()：1，4，5，同时layer[1]=layer[4]=layer[5]=2;
第三次while()：G[1]下无未访问结点，但是layer[4]=3;
第四次while()：G[4]下无为访问结点，但是layer[5]=4;layer[6]=4;
第五次while()：G[5]下未访问结点为7，本来是第三层，但因为前驱结点的layer被重复更新，此时layer[7]=5>L，orz…
